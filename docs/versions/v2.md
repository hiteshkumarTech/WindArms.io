# WindArms — v2 (Current Development Specification)

> **This is the primary specification — v2 is the current development target.** Implementation has not started yet, but every new request should be assumed to target v2 unless explicitly scoped to v1. v1 is the current stable, shipped build — see [v1.md](v1.md) — kept live and as the reference for migrating patterns (architecture, netcode, gameplay feel) into the v2 rebuild. v2 is a ground-up visual/architecture rebuild, not incremental changes on top of v1.

## Scope (from the original component-level CLAUDE.md notes)

- Hero abilities
- Matchmaking
- New UI

## Design material

The actual design specifications for v2 were originally written directly into `src/components/game/hud/CLAUDE.md` and have been moved into their own docs, preserved verbatim:

- [../design/lore.md](../design/lore.md) — world narrative ("The War Above The Storm") and the world rules abilities must obey
- [../design/art-direction.md](../design/art-direction.md) and [../design/ui.md](../design/ui.md) — visual philosophy, color palette, lighting, weapon/architecture/audio/UI direction, and the full "Master Prompt" creative-direction brief
- [../gameplay/abilities.md](../gameplay/abilities.md) — the ability-design framework (world rules, gameplay philosophy, design requirements)
- [../gameplay/operators.md](../gameplay/operators.md) — the four Founding Operators (Season Zero): per-operator kit, lore, VFX, balance, etc.

## Relationship to v1

v2 does not modify v1's code or behavior. The v1 build (all phases, engineering notes, deployment) stays exactly as documented in [v1.md](v1.md) unless and until v2 implementation work explicitly begins. Planned/undecided items that don't yet have a home in v1 or the v2 design docs above are tracked in [roadmap.md](../roadmap.md).
