# WindArms — Vision

The project's mission and core pillars. Not a roadmap (see [roadmap.md](roadmap.md)), not gameplay mechanics (see [gameplay/](gameplay/)), not lore (see [design/lore.md](design/lore.md)) — just what WindArms is trying to be, and what it must never trade away to get there.

**Read this before proposing any new feature, mechanic, or scope change.** If a proposal conflicts with a pillar below, that's a reason to stop and flag it, not a reason to quietly water down the pillar.

## Mission

Create the most satisfying browser FPS ever made.

## Core Pillars

- Every operator feels completely unique.
- No copied abilities from Valorant, Apex, Overwatch, or Deadlock.
- Competitive first.
- Browser first.
- Fast matchmaking.
- Low latency.
- High skill ceiling.

Never sacrifice these pillars.

## How this relates to existing docs

This is the first explicit statement of these pillars — earlier docs implied parts of it without stating it directly: [gameplay/abilities.md](gameplay/abilities.md)'s "Gameplay Philosophy" section ("abilities are force multipliers, not win buttons"; players win primarily through aim, movement, positioning, teamwork, timing) is the ability-design expression of "competitive first" and "high skill ceiling." [design/lore.md](design/lore.md)'s "nobody is doing this" framing is the origin of "no copied abilities." [design-principles.md](design-principles.md) is the detailed, checkable version of what these pillars mean for individual ability/mechanic design.

This file is a first pass — refine it as the team's actual priorities sharpen. If a pillar changes, log why in [decisions.md](decisions.md) rather than silently editing it.
