# WindArms — Coding Standards

> Project rules, originally in the component-level `src/components/game/hud/CLAUDE.md`, now the canonical location.

- Never remove existing features unless asked.
- Preserve responsive design.
- Prefer reusable components.
- Keep TypeScript strict.

These apply to both v1 maintenance and v2 development. See [architecture.md](architecture.md) for how the codebase is organized, and [folder-structure.md](folder-structure.md) for where new code should live.
