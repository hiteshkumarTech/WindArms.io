- **Skyfront Trial timing cleanup: gameplay timers now track real elapsed time at every frame rate; movement uses a fixed-step accumulator.** A headless low-FPS test used to verify the polish pass above surfaced a real bug: `MatchDirector.tsx` clamped its per-frame delta to `Math.min(rawDelta, 1/20)` before ticking the match clock, so below 20fps the countdown/match/respawn timers ran in slow motion (measured up to ~6-7x dilation under GPU-stalled rendering — a 3s countdown taking ~21 real seconds). Audited every clamped-delta usage across `MatchDirector`, `matchStore.tick`, `PlayerController`, `DroneSquad`, `DroneEnemy`, `DroneBoltPool`, `WindLift` and the Vortex weapon timers; only `MatchDirector`/`matchStore.tick` were bugged — `DroneEnemy`'s cooldowns/windup/stun/destroy timing and every Vortex weapon timer (fire rate, reload, inspect, ADS spin-up) were already `performance.now()`-timestamp-based, never delta-accumulated, and needed no change. Fix: `MatchDirector` now passes real per-frame delta straight through, uncapped; `matchStore.tick(realDeltaS)` itself caps a single call at `MAX_TICK_REAL_DELTA_S = 1` (exported), a documented tab-restoration policy — normal play at any frame rate is never dilated, only a genuinely large single gap (backgrounded tab) is capped rather than credited in full or ignored. New `src/lib/v2/play/fixedStep.ts` (`FIXED_STEP_S = 1/60`, `MAX_SUBSTEPS_PER_FRAME = 8`) gives `DroneSquad`/`DroneBoltPool` frame-rate-independent movement/projectile integration — bounded, no one-frame teleport, no spiral of death, full real-time-equivalent movement at 60/30/10fps, a measured-and-accepted bounded shortfall only at the extreme 5fps tier. `PlayerController` deliberately keeps its existing single clamped step (renamed `simulationDeltaS`) rather than adopting the accumulator — `lib/game/movement.ts`'s Source-style friction/accel math is not step-invariant, so multi-substep integration would be a real (if subtle) movement-feel change, which this pass's "do not change balance" constraint rules out; full reasoning in `docs/decisions.md`. New `src/lib/v2/play/matchTiming.test.ts` (wired into `npm test`, 27 new deterministic tests, 40 total passing) simulates 60/30/10/5fps for 10s of wall-clock input and asserts timers track real time exactly at every tier, pause freezes everything, the tab-restoration cap holds, and the movement accumulator stays bounded with no spiral of death. Live-verified on `/v2/play`: the match clock now reads `3:00` → `2:55` across exactly 5 real seconds. Files changed: `src/components/three/play/MatchDirector.tsx`, `PlayerController.tsx`, `DroneSquad.tsx`, `DroneBoltPool.tsx`, `WindLift.tsx`; `src/components/three/play/DroneEnemy.tsx` (param rename only); `src/lib/v2/play/matchStore.ts`, new `fixedStep.ts`, new `matchTiming.test.ts`; `package.json` (`test` script). `typecheck`/`lint`/`build`/`test` all clean.
